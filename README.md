# Meta SAM 3D Interactive Viewer

Developed as a final project for the WPI Computer Graphics course, this interactive WebGL viewer is designed to render custom 3D models generated by Meta's SAM 3D AI. 

As part of a broader Major Qualifying Project (MQP), an API was built to interact with Meta's SAM 3D to mask images and rapidly create custom 3D models for later integration into Unity. Because that API returns both Gaussian Splats (`.ply`) and traditional meshes (`.glb`), this viewer was engineered with dual rendering pipelines to toggle between and interact with both formats in real-time.

For the Gaussian Splats, the viewer implements a specialized re-lighting technique outlined by [Andrew Chan](https://andrewkchan.dev/posts/lit-splat.html). This technique uses a bilateral filter to smooth the re-lit surface and sharpens edges based on two sigmoid values, allowing for dynamic lighting on otherwise static point clouds.

## Core Features

### Dual Rendering & Texturing
* **Multi-Format Support:** Seamlessly toggles between `.glb` mesh view and `.ply` splat view.
* **Vertex Color Reconstruction:** Reconstructs vertex colors directly from the SAM 3D Gaussian Splat data. (Note: The SAM 3D source code was manually adjusted on our end to increase saturation, fixing a bug in the original model).
* **Fallback Texturing:** Custom textures are supported, and a default checkerboard texture is automatically applied if vertex colors or image textures are missing.

### Advanced Lighting & Shading
* **Dynamic Phong Shading:** The scene is illuminated by a point light and an adjustable spotlight.
* **Component Control:** Users can independently toggle the diffuse (Key `1`) and specular (Key `2`) components of the lighting model while retaining ambient light. 
* **Interactive Spotlight:** The boundary/cone of the spotlight can be adjusted in real-time using an HTML slider. 
* **Projection Shadows:** The spotlight casts a grounded projection shadow that dynamically reacts to the model's transformations and automatically disappears if the light is toggled off (Key `L` or `K`).

### Material Properties & Environment
* **Glass Refraction & Reflection:** A toggleable "Glass Mode" (Key `G`) calculates real-time reflections from the skybox and applies a 0.65 index of refraction to simulate solid glass. 
* **Textured Skybox:** The mesh environment features a fully textured, 3D skybox.

### Hierarchical Modeling & Transformations
* **Dynamic Instancing (The Hat Stack):** Users can upload a custom model (like a hat) and dynamically spawn it onto the main mesh. An infinite amount of hats can be stacked, forming a multi-level hierarchical relationship (parent, child, grandchild) where each hat inherits the scale, rotation, and translation of the model beneath it. 
* **Independent Camera Animation:** The camera operates independently from the models, allowing the user to fly around the scene using `WASD` and `E/Q`.
* **Model Transformations:** Models can be translated using arrow keys, rotated via mouse drag, and locked to the Z-axis by holding `Shift`.

## How to Use & Controls

This viewer is meant to be used in tandem with our custom 2D-to-3D SAM API. However, because that API requires a heavy cloud GPU and Docker image, we have provided test models in the `/testModels` directory so you can evaluate the viewer locally. (We highly recommend testing with the `sonic` generation!).

**Camera & Movement:**
* `W` `A` `S` `D` - Translate Camera (Forward, Left, Back, Right)
* `E` / `Q` - Translate Camera (Up / Down)
* `Arrow Keys` - Translate the 3D Model
* `Mouse Drag` - Rotate Model (X/Y Axis)
* `Shift + Mouse Drag` - Rotate Model (Z-Axis Lock)

**Lighting & Materials:**
* `L` - Toggle Main Light On/Off
* `K` - Toggle Projection Shadows On/Off
* `G` - Toggle Glass Reflection/Refraction Mode (Mesh Only)
* `1` - Toggle Diffuse Lighting
* `2` - Toggle Specular Highlights

## Troubleshooting & Loading Issues

**The "Invisible Hat" Issue:**
If you click the "Spawn Hat" button and nothing appears on your model, it means the viewer doesn't have a hat model loaded into memory yet! Make sure you use the HTML file upload menu to upload your secondary `.glb` file before clicking the spawn button. 

**CORS Network Errors (Blank Screen):**
If you open this project by double-clicking the `index.html` file and get a console error like `Uncaught (in promise) TypeError: NetworkError when attempting to fetch resource`, your browser's CORS security policy is blocking the local file fetch. 
* **Fix 1:** Open the project using an IDE like WebStorm and use their built-in browser preview.
* **Fix 2:** Run a local web server (e.g., `python3 -m http.server`) in the project directory and view it at `http://localhost:8000`.
* **Fix 3:** Simply use the HTML "Upload Models" buttons on the page to manually inject the test models into the scene, which bypasses the CORS fetch restriction entirely.

## Technical Challenges

There were many engineering challenges faced during this project, heavily centered around rendering the Gaussian Splats:
1. **Splat Parsing:** Existing loaders for rendering splats are incredibly heavy. Finding and implementing a lightweight method to accurately parse the raw `.ply` binary data was a major hurdle.
2. **WebGL 2.0 Transition:** The relighting environment and splats required upgrading to WebGL 2.0. While this presented a learning curve, it ultimately provided cleaner matrix handling and built-in functions (like `inverse()` and `refract()`) that were vital for our glass shaders.
3. **Bilateral Filtering:** Implementing Andrew Chan's splat relighting required utilizing complex framebuffers and understanding pre/post-processing textures to properly calculate the spatial and range weights for the blur.
4. **Dual Rendering Pipelines:** Housing both the mesh and splat pipelines in the exact same scene required careful state management, complex conditionals in our JavaScript, and maintaining entirely separate WebGL shader programs within the same HTML file to prevent the environments from conflicting.

Future implementation will involve integrating the actual 2D-to-3D API directly within the JavaScript so objects could be generated and fetched in real-time without needing a separate application to trigger the generation.
